# 環境プリフライト

## セッション開始時（最初の作業前に1回）

作業に入る前に以下を確認し、結果を1行で報告する:

1. `pwd` — 想定リポジトリにいるか（マルチリポ構成では特に必須）
2. `git branch --show-current` — 想定ブランチか。main 直上での作業開始は警告
3. ファイル数突合 — `find` 系探索の結果が `git ls-files` の数と大きく乖離していないか（iCloud 等のクラウド退避でローカルからファイルが消えていると、Claude から不可視になり修正漏れ→CI失敗の原因になる。乖離があればユーザーに報告して作業を止める）

## git 操作の直前

commit / push / branch 切替 / PR 作成の前に pwd と branch を再確認する。
特にマルチリポ・複数 worktree 環境では、直前の cd がどこを向いているかを暗黙の前提にしない。

## 並行作業への配慮

- ユーザーがローカルで検証作業中の可能性がある場合、DB への書き込み・ブランチ切替・サーバー再起動の前に確認を取る
- ローカル検証と並行する実装作業は worktree に分離する（parallel-work.md 参照）
