# プロジェクト規模の閾値定義

## 規模チェック方法
```bash
find . -type f -not -path './.git/*' | wc -l
```

## 閾値一覧

| 閾値 | 単位 | 用途 | 参照スキル |
|------|------|------|-----------|
| 200 | ファイル | コード分析の前処理要否判定。超過時は map が内蔵 Explore を並列起動 | map |
| 500 | ファイル | 骨格分析の閾値。超過時はサンプリング分析に切り替え | map |
| 3,000 | 行（1ファイル） | ドメイン分割の目安。超過するファイルは分割を検討 | map |
| 6 | 機能/ドメイン | 1ドメインあたりの機能数上限。超過時はドメイン分割を提案 | plan |
| 20 | 要件カード数 | フラット管理 vs カテゴリ分類の切替点。超過時はカテゴリ別ディレクトリに整理 | grill |

## 閾値: 200ファイル（コード分析）

200ファイルを超えると、単一エージェントが全ファイルを読みながら横断的な関連を把握する精度が低下する転換点となる。

## 判定ロジック
| ファイル数 | アクション |
|-----------|-----------|
| ≤200 | 本体が直読みで把握できる。そのまま続行 |
| >200 | map が内蔵 Explore を並列起動して分担分析する |
