# ドメイン知識ベース

プロジェクト固有の用語・テーブル定義・仕様の「調べ直しが発生しがちな知識」を、
`file:line` 付きで蓄積する場所。**毎セッションで同じ調査を繰り返さないため**の共有資産。

## 目的

- 同じドメイン知識を別々のセッション・別々のメンバーが繰り返し再調査する無駄をなくす
- Claude が調査時に最初に参照する一次情報源にする（記憶や推測ではなく、ここの `file:line` を引く）
- 用語の初手誤解（テーブル名・概念の取り違え）を防ぐ

## 使い方

1. 調査で判明した「次も使う」知識を `qa-template.md` の形式で追記する
2. 必ず根拠を `path/to/file.ts:42` 形式で残す（コードが動く一次情報源を指す）
3. コードが変わって記述が古くなったら、調査したついでに更新する（鮮度はメンテで保つ）
4. テーマ別にファイルを分けてよい（例: `billing.md`, `reservation.md`, `auth.md`）

## 運用ルール

- **記憶で答えず、ここを引く** — ドメインの質問はまずこのディレクトリを Read する（エビデンス検証の原則）
- **重複調査を見つけたらここへ昇格** — 2回以上調べたことは、ここに書いて以後の再調査をゼロにする
- 各エントリは「質問 → 結論 → 根拠（file:line）→ 補足」の最小構成を守る
