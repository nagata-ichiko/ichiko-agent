## 1. 目的

実装中に開発者へ質問が発生する残存3経路（grill 問い詰め表のカバレッジ外・既存コードとの矛盾の遅延発覚・Fixゲートの自己申告）を塞ぎ、「一度任せたら後から不明点が出ない」を強化する。

## 2. ユーザーストーリー

- 開発者として、要件確定時点で技術前提（外部連携・環境・エラー方針）まで問い詰められたい。なぜなら実装中に質問で中断されたくないから
- 開発者として、既存コードと要件の矛盾は grill 中に指摘されたい。なぜなら build 開始後の差し戻しは手戻りが大きいから
- 開発者として、Fixゲートの達成は機械判定されたい。なぜならAIの自己申告は信用の根拠にならないから
