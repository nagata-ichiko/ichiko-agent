# 着手前のスコープ復唱（Go承認）

実装・削除・リファクタ・文書の大幅変更に着手する前に、以下を提示してユーザーの Go を待つ。

1. **理解した依頼内容を3行以内で復唱**
2. **触るファイル一覧**（新規作成を含む）
3. **曖昧語の解釈** — 「通知」「削除」「対応」など複数解釈がありうる語は、採用した解釈を明示する
4. **やらないこと** — スコープ外と判断した範囲

## 例外

- 誤字修正など1〜2ファイル・数行の自明な修正
- ユーザーが「確認不要で進めて」と明示した場合
- PlanMode・implement-spec 等で実装プランを提示する場合は、プラン提示がスコープ復唱を兼ねる（曖昧語の解釈は必ずプランに含める）

## 禁止

- 復唱を提示した直後に、ユーザーの応答を待たずに編集を開始すること
- 曖昧語を含む依頼を、解釈を明示しないまま単一の解釈で進めること
